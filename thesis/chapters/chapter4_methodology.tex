\chapter{Methodology}
\label{ch:methodology}

This chapter outlines the way I established the experiments. I present a formal statement of the classification problem, how the symptoms are represented as features, each of the five classifiers I selected and why, and finally, how to deal with class imbalances, the tuning of the hyperparameters, and the evaluation metrics.

\section{Problem Formulation}
\label{sec:problem_formulation}

This is multi-class supervised classification. With a patient symptom vector, say, of the form, so, we want to predict the disease label, say, of the form, so, where, in turn, is the number of distinct symptoms, and is the number of disease categories.

Formally, I want to learn a mapping $f: \{0, 1\}^d \rightarrow \{1, \ldots, C\}$ that minimizes classification error on unseen patients. I train the classifiers on a labeled training set $\mathcal{D} = \{(\mathbf{x}_i, y_i)\}_{i=1}^{N}$ and evaluate on a held-out test set.

This formulation makes three simplifying assumptions:
\begin{enumerate}
    \item The symptom representation is the same whether the symptom was self-reported or clinically observed.
    \item The datasets reasonably represent real symptom-disease patterns, though they're obviously simplified compared to actual clinical encounters.
    \item Each patient has exactly one disease label. In reality, patients often have multiple conditions simultaneously, but the datasets don't support multi-label classification.
\end{enumerate}

\section{Feature Representation}
\label{sec:feature_representation}

Each patient is represented as a binary vector of dimension $d = 132$ (for Dataset~1). For example, a patient with fever, headache, and nausea would have:

\begin{equation}
\label{eq:feature_vector}
\mathbf{x} = [\underbrace{0, \ldots, 0}_{\text{other symptoms}}, \underbrace{1}_{\text{fever}}, \underbrace{0, \ldots, 0}_{\text{other symptoms}}, \underbrace{1}_{\text{headache}}, \underbrace{0, \ldots, 0}_{\text{other symptoms}}, \underbrace{1}_{\text{nausea}}, \underbrace{0, \ldots, 0}_{\text{other symptoms}}],
\end{equation}

\noindent with 1s at the location of the reported symptoms and 0s at all other locations.

This works like the bag-of-words in text classification. It records the presence of the symptoms but nothing about the severity, duration, and order. That is an actual limitation, though in practice binary symptom vectors can be surprisingly useful in the disease classification task, particularly when the classifier is able to identify combinations of features.

\section{Classification Models}
\label{sec:models}

I chose five classifiers which occupy the spectrum of simple to complex. The criteria were: (1) it must have a defined use in medical classification, (2) the collection must span the various algorithmic methods (probabilistic, linear, kernel-based, ensemble) and (3) all should have full-fledged implementations in scikit-learn or XGBoost.

\subsection{Naive Bayes}
\label{sec:naive_bayes}

Naive Bayes was used as the simplest baseline. It uses Bayes theorem when all the features are independent conditionally on the class of the example \citep{Bishop2006}. I adopt the Bernoulli variant because the characteristics are binomial:

\begin{equation}
\label{eq:naive_bayes}
P(y \mid \mathbf{x}) \propto P(y) \prod_{j=1}^{d} P(x_j \mid y).
\end{equation}

The independence assumption is obviously false, the co-occurrence of fever and headache is always present. However, I added Naive Bayes since it trains faster than a second and provides an indication of a floor against which the more sophisticated algorithms must perform. Implementation: \texttt{sklearn.naive\_bayes.BernoulliNB}.

\subsection{Logistic Regression}
\label{sec:logistic_regression}

The second model that I chose was the Logistic Regression, primarily due to its interpretability. It represents the probabilities of classes as a softmax of linear features \citep{Bishop2006):

\begin{equation}
\label{eq:logistic_regression}
P(y = k \mid \mathbf{x}) = \frac{\exp(\mathbf{w}_k^T \mathbf{x} + b_k)}{\sum_{j=1}^{C} \exp(\mathbf{w}_j^T \mathbf{x} + b_j)}.
\end{equation}

I used L2 regularization to prevent overfitting. The reason I kept this one is that the learned weights $\mathbf{w}_k$ directly show which symptoms push toward which disease --- if a doctor asks ``why did it recommend a dermatologist?'', Logistic Regression can give a clear answer. Implementation: \texttt{sklearn.linear\_model.LogisticRegression} with \texttt{lbfgs}.

\subsection{Support Vector Machine}
\label{sec:svm}

SVM was included to test a margin-based classifier \citep{Cortes1995}. For Dataset~1, I used an RBF kernel because the dataset is small enough for a full kernel method:

\begin{equation}
\label{eq:svm_kernel}
K(\mathbf{x}_i, \mathbf{x}_j) = \exp\left(-\gamma \|\mathbf{x}_i - \mathbf{x}_j\|^2\right).
\end{equation}

For Dataset~2, a full kernel SVM would be too slow, so I used a linear approximation based on stochastic gradient descent with hinge loss. This keeps the model practical on the larger dataset while still preserving the basic SVM objective. Implementation: \texttt{sklearn.svm.SVC} for Dataset~1 and \texttt{sklearn.linear\_model.SGDClassifier} for Dataset~2.

\subsection{Random Forest}
\label{sec:random_forest}

Random Forest was the first ensemble method I tested \citep{Breiman2001}. It builds many trees on bootstrap samples, each seeing only a random subset of features at each split:

\begin{equation}
\label{eq:random_forest}
\hat{y} = \text{mode}\{h_1(\mathbf{x}), h_2(\mathbf{x}), \ldots, h_T(\mathbf{x})\}.
\end{equation}

I chose this because it handles binary data well, needs no feature scaling, and gives feature importance scores (which I use later in Section~\ref{sec:feature_importance}). In early tests, 200 trees already gave solid results and trained in just a few seconds. Implementation: \texttt{sklearn.ensemble.RandomForestClassifier}.

\subsection{XGBoost}
\label{sec:xgboost}

XGBoost builds trees sequentially --- each new tree corrects the mistakes of previous ones \citep{Chen2016}:

\begin{equation}
\label{eq:xgboost}
\mathcal{L}^{(t)} = \sum_{i=1}^{N} l(y_i, \hat{y}_i^{(t-1)} + f_t(\mathbf{x}_i)) + \Omega(f_t).
\end{equation}

I included XGBoost mainly to see if its extra complexity over Random Forest actually buys anything for symptom data. It has a reputation for dominating Kaggle competitions on tabular datasets. The regularization term $\Omega$ should help with the smaller Dataset~1 where overfitting is more of a risk. Implementation: \texttt{xgboost.XGBClassifier}.

\section{Class Imbalance Handling}
\label{sec:class_imbalance}

Both datasets have class imbalance --- some diseases appear much more often than others. Without handling this, classifiers tend to over-predict common diseases and ignore rare ones.

I used two strategies:

\textbf{Class Weighting:} For Logistic Regression, SVM, and Random Forest, I set \texttt{class\_weight='balanced'}, which adjusts each class's weight inversely to its frequency:

\begin{equation}
\label{eq:class_weight}
w_k = \frac{N}{C \cdot n_k},
\end{equation}

\noindent where $N$ denotes the total sample size, $C$ the number of classes and $n_k$ the size of class $k$. This implies that rare diseases are more severely punished in the loss function at the time of misclassification.

\textbf{Stratified Sampling:} All train-test splits and cross-validation folds are stratified to maintain class proportions.

I have thought of SMOTE \citep{Chawla2002} but dismissed it to use in this application. The interpolation-based generation of synthetic examples of SMOTE is discussed in problematic terms with binary feature spaces: averaging interpolates between binary symptom vectors generate fractional values that are not possible symptom presentations \citet{Fernandez2018}. Class weighting has similar balancing effects, but it does not create artificial data points.

\section{Hyperparameter Tuning}
\label{sec:hyperparameter_tuning}

I optimized hyperparameters by grid search with stratified 5-fold cross-validation on the training data of Dataset 1. Table~\ref{tab:hyperparameters} in Appendix~\ref{app:supplementary} contains the search spaces. Dataset2 is operated with fixed settings, which have been selected to make the runtime manageable.

The scoring metric that I employed in grid search was macro-averaged F1-score rather than accuracy. This is because a classifier can give misleadingly high accuracy on data which is not balanced, e.g. a classifier which predicts the most frequent disease may achieve 40 percent accuracy. Macro-F1 uses all the classes equally, and this is what compels the tuning process to be concerned about the performance of the rare-diseases, rather than simply optimizing the majority of the classes.

\section{Evaluation Metrics}
\label{sec:evaluation_metrics}

I perform an evaluation of each classifier on four measures, which are calculated on the held-out test set.

\textbf{Accuracy} The fraction of correct predictions:

\begin{equation}
\label{eq:accuracy}
\text{Accuracy} = \frac{\text{Number of correct predictions}}{\text{Total number of predictions}}.
\end{equation}

\textbf{Precision} measures how many of the predicted positives are actually correct for each class $k$:

\begin{equation}
\label{eq:precision}
\text{Precision}_k = \frac{TP_k}{TP_k + FP_k}.
\end{equation}

\textbf{Recall} measures how many of the actual positives are correctly identified:

\begin{equation}
\label{eq:recall}
\text{Recall}_k = \frac{TP_k}{TP_k + FN_k}.
\end{equation}

\textbf{F1-score} is the harmonic mean of precision and recall:

\begin{equation}
\label{eq:f1}
\text{F1}_k = 2 \cdot \frac{\text{Precision}_k \cdot \text{Recall}_k}{\text{Precision}_k + \text{Recall}_k}.
\end{equation}

I report macro-averaged versions of precision, recall, and F1-score:

\begin{equation}
\label{eq:macro}
\text{Macro-F1} = \frac{1}{C} \sum_{k=1}^{C} \text{F1}_k.
\end{equation}

I selected macro-averaging, as opposed to micro-averaging, as all disease classes are treated equally. Under micro-averaging, prevalent diseases will take up most of the score and conceal low scores on rare diseases. In the case of a healthcare application, it would be more appropriate to have equal weights since missing a disease that is rare and severe is more severe than misclassifying a common cold.

I also created confusion matrices of the most performing classifier to understand which diseases are confused with each other.
