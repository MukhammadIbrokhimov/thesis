\chapter{Methodology}
\label{ch:methodology}

This chapter presents the methodology used to compare the five machine learning classifiers. It begins with the formal problem definition, describes the feature representation, details each classifier and the rationale for its inclusion, and concludes with the evaluation framework.

\section{Problem Formulation}
\label{sec:problem_formulation}

The task addressed in this thesis is multi-class supervised classification. Given a patient symptom vector $\mathbf{x} \in \{0, 1\}^d$, where $d$ is the number of distinct symptoms and each element indicates the presence (1) or absence (0) of a symptom, the goal is to predict the disease label $y \in \{1, 2, \ldots, C\}$, where $C$ is the number of disease categories.

Formally, the objective is to learn a mapping $f: \{0, 1\}^d \rightarrow \{1, \ldots, C\}$ that minimizes the expected classification error over unseen patient records. The classifiers are trained on a labeled training set $\mathcal{D} = \{(\mathbf{x}_i, y_i)\}_{i=1}^{N}$ and evaluated on a held-out test set.

The following assumptions underlie this formulation:
\begin{enumerate}
    \item Symptoms are independent of the observation method --- the same symptom representation is used regardless of how the symptom was reported.
    \item The datasets are representative of the underlying symptom-disease distribution, acknowledging that publicly available datasets may not capture the full complexity of clinical encounters.
    \item The disease labels are correct and unambiguous, which is a simplification given that real-world diagnoses often involve uncertainty.
\end{enumerate}

\section{Feature Representation}
\label{sec:feature_representation}

Each patient record is represented as a binary feature vector of dimension $d = 132$ (for Dataset~1). Consider a patient presenting with fever, headache, and nausea. The corresponding feature vector would be:

\begin{equation}
\label{eq:feature_vector}
\mathbf{x} = [\underbrace{0, \ldots, 0}_{\text{other symptoms}}, \underbrace{1}_{\text{fever}}, \underbrace{0, \ldots, 0}_{\text{other symptoms}}, \underbrace{1}_{\text{headache}}, \underbrace{0, \ldots, 0}_{\text{other symptoms}}, \underbrace{1}_{\text{nausea}}, \underbrace{0, \ldots, 0}_{\text{other symptoms}}],
\end{equation}

\noindent where only the positions corresponding to the three reported symptoms have a value of 1 and all remaining positions are 0.

This binary bag-of-symptoms representation is analogous to the bag-of-words model in natural language processing. It captures which symptoms are present but does not encode symptom severity, temporal ordering, or co-occurrence patterns beyond what the classifiers can learn from the feature combinations. Despite this simplicity, binary symptom vectors have been shown to be effective for disease classification when combined with appropriate classifiers \citep{Garg2022}.

\section{Classification Models}
\label{sec:models}

Five classifiers were selected for comparison, spanning a range of model complexities and learning paradigms. The selection was guided by three criteria: (1) established use in medical classification literature, (2) diversity of underlying algorithmic approaches, and (3) availability of efficient implementations in scikit-learn and XGBoost libraries.

\subsection{Naive Bayes}
\label{sec:naive_bayes}

Naive Bayes is a probabilistic classifier based on Bayes' theorem with the strong assumption that features are conditionally independent given the class label \citep{Bishop2006}. For binary symptom data, the Bernoulli Naive Bayes variant is used, which models each feature as an independent Bernoulli random variable:

\begin{equation}
\label{eq:naive_bayes}
P(y \mid \mathbf{x}) \propto P(y) \prod_{j=1}^{d} P(x_j \mid y).
\end{equation}

\textbf{Justification:} Naive Bayes serves as a strong baseline due to its simplicity, fast training time, and surprisingly competitive performance on high-dimensional data. Its conditional independence assumption, while unrealistic for medical symptoms (which often co-occur), provides a useful lower bound for comparison with more sophisticated methods.

\subsection{Logistic Regression}
\label{sec:logistic_regression}

Logistic Regression is a linear classifier that models the posterior probability of each class as a softmax function of a linear combination of features \citep{Bishop2006}. For multi-class classification, the one-vs-rest (OvR) strategy is used:

\begin{equation}
\label{eq:logistic_regression}
P(y = k \mid \mathbf{x}) = \frac{\exp(\mathbf{w}_k^T \mathbf{x} + b_k)}{\sum_{j=1}^{C} \exp(\mathbf{w}_j^T \mathbf{x} + b_j)},
\end{equation}

\noindent where $\mathbf{w}_k$ and $b_k$ are the weight vector and bias for class $k$.

\textbf{Justification:} Logistic Regression provides an interpretable linear baseline. Its coefficients directly indicate the contribution of each symptom to each disease prediction, making it valuable in healthcare contexts where model transparency is important. L2 regularization is applied to prevent overfitting on high-dimensional symptom vectors.

\subsection{Support Vector Machine}
\label{sec:svm}

Support Vector Machine (SVM) finds the hyperplane that maximizes the margin between classes in the feature space \citep{Cortes1995}. For non-linearly separable data, the radial basis function (RBF) kernel is used to project features into a higher-dimensional space:

\begin{equation}
\label{eq:svm_kernel}
K(\mathbf{x}_i, \mathbf{x}_j) = \exp\left(-\gamma \|\mathbf{x}_i - \mathbf{x}_j\|^2\right),
\end{equation}

\noindent where $\gamma$ controls the kernel width. Multi-class classification is handled using the one-vs-one strategy.

\textbf{Justification:} SVM is included because of its strong theoretical foundations and effectiveness in high-dimensional spaces with binary features. The RBF kernel allows it to capture non-linear symptom interactions that linear models cannot represent.

\subsection{Random Forest}
\label{sec:random_forest}

Random Forest is an ensemble method that constructs multiple decision trees during training and outputs the mode of their predictions \citep{Breiman2001}. Each tree is trained on a bootstrap sample of the data, and at each split, only a random subset of features is considered:

\begin{equation}
\label{eq:random_forest}
\hat{y} = \text{mode}\{h_1(\mathbf{x}), h_2(\mathbf{x}), \ldots, h_T(\mathbf{x})\},
\end{equation}

\noindent where $h_t$ denotes the $t$-th decision tree and $T$ is the total number of trees.

\textbf{Justification:} Random Forest is robust to overfitting, handles high-dimensional data well, and provides built-in feature importance estimates. It does not require feature scaling and can naturally capture complex interactions between symptoms. It serves as the primary representative of bagging-based ensemble methods.

\subsection{XGBoost}
\label{sec:xgboost}

XGBoost (Extreme Gradient Boosting) is a gradient boosting framework that builds an ensemble of decision trees sequentially, where each new tree corrects the errors of the previous ensemble \citep{Chen2016}. The objective function at iteration $t$ is:

\begin{equation}
\label{eq:xgboost}
\mathcal{L}^{(t)} = \sum_{i=1}^{N} l(y_i, \hat{y}_i^{(t-1)} + f_t(\mathbf{x}_i)) + \Omega(f_t),
\end{equation}

\noindent where $l$ is the loss function, $f_t$ is the new tree, and $\Omega(f_t)$ is a regularization term that penalizes model complexity.

\textbf{Justification:} XGBoost represents the state of the art in gradient boosting and has consistently achieved top results in machine learning competitions and medical classification studies \citep{Chen2016}. Its regularization mechanisms and efficient implementation make it suitable for both small and large datasets. It serves as the primary representative of boosting-based ensemble methods.

\section{Class Imbalance Handling}
\label{sec:class_imbalance}

Both datasets exhibit class imbalance, where some disease categories have significantly more examples than others. This imbalance can bias classifiers toward predicting majority classes, resulting in poor recall for rare diseases.

Two complementary strategies are employed to address this issue:

\textbf{Class Weighting:} For classifiers that support it (Logistic Regression, SVM, Random Forest, XGBoost), the \texttt{class\_weight='balanced'} parameter is used. This automatically adjusts the weight of each class inversely proportional to its frequency:

\begin{equation}
\label{eq:class_weight}
w_k = \frac{N}{C \cdot n_k},
\end{equation}

\noindent where $N$ is the total number of samples, $C$ is the number of classes, and $n_k$ is the number of samples in class $k$.

\textbf{Stratified Sampling:} All train-test splits and cross-validation folds use stratified sampling to ensure consistent class proportions, preventing evaluation artifacts caused by uneven class distributions across folds.

SMOTE \citep{Chawla2002} was considered but not applied, as it can introduce noise when interpolating between binary feature vectors and its effectiveness on high-dimensional binary data is limited \citep{Fernandez2018}.

\section{Hyperparameter Tuning}
\label{sec:hyperparameter_tuning}

Hyperparameters for each classifier were tuned using grid search with stratified 5-fold cross-validation on the training set. The search space for each classifier was defined based on commonly used ranges in the literature and preliminary experiments. The complete hyperparameter search spaces and the selected optimal values are documented in Table~\ref{tab:hyperparameters} in Appendix~\ref{app:supplementary}.

The scoring metric used for hyperparameter selection was the macro-averaged F1-score, which gives equal weight to all classes regardless of their frequency. This choice ensures that the selected hyperparameters optimize performance across all disease categories, including rare ones, rather than favoring majority classes.

\section{Evaluation Metrics}
\label{sec:evaluation_metrics}

The classifiers are evaluated using four standard metrics for multi-class classification. All metrics are computed on the held-out test set and reported as both per-class and macro-averaged values.

\textbf{Accuracy} measures the proportion of correctly classified instances:

\begin{equation}
\label{eq:accuracy}
\text{Accuracy} = \frac{\text{Number of correct predictions}}{\text{Total number of predictions}}.
\end{equation}

\textbf{Precision} measures the proportion of predicted positives that are true positives for each class $k$:

\begin{equation}
\label{eq:precision}
\text{Precision}_k = \frac{TP_k}{TP_k + FP_k},
\end{equation}

\noindent where $TP_k$ and $FP_k$ denote true positives and false positives for class $k$.

\textbf{Recall} (sensitivity) measures the proportion of actual positives that are correctly identified:

\begin{equation}
\label{eq:recall}
\text{Recall}_k = \frac{TP_k}{TP_k + FN_k},
\end{equation}

\noindent where $FN_k$ denotes false negatives for class $k$.

\textbf{F1-Score} is the harmonic mean of precision and recall, providing a balanced measure:

\begin{equation}
\label{eq:f1}
\text{F1}_k = 2 \cdot \frac{\text{Precision}_k \cdot \text{Recall}_k}{\text{Precision}_k + \text{Recall}_k}.
\end{equation}

The macro-averaged versions of precision, recall, and F1-score are computed by averaging the per-class values:

\begin{equation}
\label{eq:macro}
\text{Macro-F1} = \frac{1}{C} \sum_{k=1}^{C} \text{F1}_k.
\end{equation}

Macro-averaging is preferred over micro-averaging in this context because it treats all disease classes equally, ensuring that the evaluation is not dominated by frequent diseases. This aligns with the clinical requirement that rare diseases should be predicted as reliably as common ones.

Additionally, confusion matrices are generated for each classifier to visualize the pattern of misclassifications and identify which disease pairs are most commonly confused.
